Adding a Search Bar to Your Website

It has been 4 months since this blog has started and the look of it has not really changed much, especially the header of the page. And ever since the start, there has been that mysterious search bar at the top right that has done absolutely nothingâ€¦ Until now! You can type anything you want into the bar and press Enter or the magnifying glass and it will bring you to a search results page. I set it to filter based on the title, description, tags, and some keywords. In this post, I will show you how you can setup your own search bar on your websites and customize them however you want!

The first thing you will need is a website (or at least the start of one). Ideally, this should be a site with multiple pages so your search feature can work across the pages. A perfect example is a blog website but any type of website can work also. Next, you will need somewhere to place your search bar. Search bars are usually placed on the top right of pages so if you have a header, you can align your search bar to the right. Here is an example header you can have on your pages.

Next, you will want to add an onclick() attribute to your search button. Note that it does not matter what type of element it is (button, span, div, etc.), the onclick would work for it regardless. You can name the function whatever you wish, I will use search(). Then, you must make sure you have included the script where you will handle the JavaScript logic. Here is the updated section of code:

 <!-- Search Bar -->
 <div class="search-bar">
  <input type="text" placeholder="Search...">
  <button type="button" onclick="search()">Search</button>
 </div>

 <script src="search.js"></script
</body>
</html>

Now is where you will start to add the logic to the search feature. Before doing so, there is a decision you will make. The easiest way to handle the content that gets searched through is to save it in some way. You can either save it within the html/js files, or you can save it in a JSON file. I believe the second option is better as it is easier to edit and manage. The first option is valid if you are making a website that will not be hosted at all (locally or online) as if that is the case, you cannot read through JSON files.

There are also other ways you can create a search feature such as using the Fetch API with JavaScript, or AJAX libraries. In my opinion though, using a JSON file can be simpler because you can control what you want to search and it is very easy to edit and make changes. For the other methods, since you are accessing the html content directly from other pages, it can cause issues if your structure or content were to ever change. And for the JSON file, you can easily set up a script to read your html files and generate the JSON file according to that.

Now into the functionality of the search. Before getting into the search, you can add an event listener to your search bar that checks if Enter is pressed to also allow users to begin the search with that. Next, in your search() function, you will want to capture the input that is written in the search bar and then redirect to the html page that will contain the search results.

function search() {
 console.log('Searching...');

 var query = document.getElementById('searchInput').value;
 if (query) {
  // Redirect to search-results.html with the query as a URL parameter
  window.location.href = 'search-results.html?q=' + encodeURIComponent(query);
 }
 return false;
}

document.getElementById('searchInput').addEventListener('keypress', function(event) {
 if (event.key === 'Enter') {
  event.preventDefault();
  search();
 }
});

Now we must display the search results in a new html page. For this, we can create a simple <div> that will display what we searched, and hold the contents of the search result. To display the contents, we must add some JavaScript to the search results page. We must first get the search query value, then get the content from the JSON file, and then filter the results based on the query. Before we get to that, we have to set the layout for our JSON file. This is where you have the freedom to decide what you want to be involved in the searching functionality. For my search feature, I include the title, description, tags, and keywords. You can add more such as the content, image names, etc. This is what some of my JSON file looks like:

Now for the search functionality, we have our three functions: Get the query, fetch the JSON file, and render the results. For getting the query, since we passed the parameters in our link when we redirected, we can easily access those using the following:

// Function to parse query from URL
function getQueryParameter(param) {
 var urlParams = new URLSearchParams(window.location.search);
 return urlParams.get(param);
}

For fetching our JSON data, we can use a simple try catch function to attempt to access it and return an error if it is not found or if it cannot access them:

// Function to fetch posts data from JSON file
async function fetchPosts() {
 try {
  const response = await fetch('posts.json');
  if (!response.ok) {
   throw new Error('Failed to load posts');
  }
  return await response.json();
 } catch (error) {
  console.error('Error fetching posts:', error);
  return [];
 }
}

For rendering our results, here is where the code will differ. Here is mine for reference:

// Function to render search results
function renderResults(query, posts) {
 const resultsContainer = document.getElementById('results-container');
 const searchQuerySpan = document.getElementById('search-query');
 searchQuerySpan.textContent = query;

 // Filter posts by title, description, tags, or keywords matching the query
 const filteredPosts = posts.filter(post =>
  post.title.toLowerCase().includes(query.toLowerCase()) ||
  post.description.toLowerCase().includes(query.toLowerCase()) ||
  post.tags.some(tag => tag.toLowerCase().includes(query.toLowerCase())) ||
  post.keywords.some(keyword => keyword.toLowerCase().includes(query.toLowerCase()));

 // Display results or no results message
 if (filteredPosts.length > 0) {
  filteredPosts.forEach(post => {
   const postHtml = `
    `<div class="card">
     <a href="${post.link}">
     <img class="img-fluid" src="${post.image}" alt="">
     </a>
    <div class="card-block">
     <h2 class="card-title"><a href="${post.link}">${post.title}</a></h2>
     <h4 class="card-text">${post.description}</h4>
     <div class="metafooter">
      <div class="wrapfooter">
       <span class="author-meta">
        <span class="post-date">${post.date}</span>
        </span>
      </div>
     </div>
    </div>`;
    resultsContainer.innerHTML += postHtml;
   });
 } else {
  resultsContainer.innerHTML = '<p>No results found for your search.</p>';
 }
}

The start simply gets the search query and prepares the div to display the results, Then, depending on how you structure your JSON format and what you choose to filter, the const filteredPosts will be different, but by looking at my format, you can easily figure out how it is done. Just edit the second variable of the following line to be the same name as you gave in the name section of your JSON file. In this case, you would change title:

post.title.toLowerCase().includes(query.toLowerCase())

Next, is where things will also be different. This all depends on how the layout and CSS is of your webpage. For mine, they are styled by using cards within Bootstrap that can be quickly setup by searching online or using tools like ChatGPT to generate them for you. Within the html, you can embed the content of your results however you would like. Finally, at the end you can add an eventlistener to run all the necessary functions when the page is loaded to get your search results right away.

// Initialize search results page
document.addEventListener("DOMContentLoaded", async function() {
 const query = getQueryParameter('q');
 if (query) {
  const posts = await fetchPosts();
  renderResults(query, posts);
 } else {
  document.getElementById('results-container').innerHTML = '<p>Please enter a search query.</p>';
 }
});

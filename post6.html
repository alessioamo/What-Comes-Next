How to Create a Discord Bot

The year is 2016, you are scrolling through Facebook when you hear the signature Skype ringtone as a popup appears on your screen. You answer and your friend starts talking about a new app to use where you can talk to each other instead of using Skype. "What's it called?" you ask. He sends you a link and says "Discord".

Discord is the most popular gaming communication application and one of the most popular communication applications in general. From its ease of use, to its customization, to its communities and more. One thing that puts it apart from the rest of the applications is the use of bots. Discord has an API and a lot of documentation and support for users to create and access bots for their servers. These bots can range from simple bots like replying to messages with gifs, to very complicated bots that manage entire servers. In this post, I will show you how to create your own bot with custom commands and how to host it.

First, you need to head to the Discord Development Page and login to your Discord account. Then, click on New Application at the top right of the page. Give a name to your application and confirm it. This will take you to a new page with a sidebar and many options. Select Bot and if there is not already one created (if not there will be nothing under Build-A-Bot), click on the button at the top right that says Add Bot.

Here you can customize your bot by giving it a username, profile picture, and banner. Before proceeding, you should see a token under username (a long line of random characters). Copy this and store it somewhere safe because it will disappear the next time you visit this page. Also, make sure to never share this with anyone because then they could access your bot and alter it.

Next, we must invite your bot to your server(s). Generally, I create a test server for bot testing and invite my bot there first before inviting it to other servers. To generate an invite link, go to OAuth2 in the sidebar right above Bot. If you scroll down you will see OAuth2 URL Generator. In this section, select Bot and a new section with bot permissions will show up. This is where you can select what permissions you want your bot to have like sending messages, reacting to messages, etc. It is recommended to give this bot admin privileges to make it easier to work with but that is always up to you.

Make sure Integration Type is on Guild Install and then copy the link. Paste it into your browser and select the server(s) you would like to add your bot to. Now, we will get into the coding of the bot. Before starting, you will need VSCode and Node.js installed on your computer. Once complete, open VSCode and create a new folder for you to work in. Open your Terminal by selecting Terminal -> New Terminal or using the shortcut Ctrl + `.

You can check if Node.js installed properly by entering
node -v and
npm -v. Next, run
npm init -y and once complete, run
npm i discord.js. These will download all the proper libraries that you will need. Next, create a file index.js and a file config.json. In config.json, you will want to put: 


{
    "token": "PASTE TOKEN HERE"
}


And replace PASTE TOKEN HERE with that token you copied earlier. In index.js, you can write: 


const { token } = require('./config.json');
const {
    Client,
    GatewayIntentBits,
    ButtonBuilder,
    ButtonStyle,
    ModalBuilder,
    TextInputBuilder,
    TextInputStyle
} = require('discord.js');

const client = new Client ({
    intents: [
        GatewayIntentBits.Guilds,
        GatewayIntentBits.GuildMessages,
        GatewayIntentBits.GuildMembers,
        GatewayIntentBits.DirectMessages,
        GatewayIntentBits.MessageContent,
        GatewayIntentBits.GuildMessageReactions
    ]
});

client.login(token);

client.on("messageCreate", async (message) => {

    if (!message?.author.bot) {
        message.reply("Hello");
    }

});


This will make it so your bot will reply to any message sent in the server that is not from a bot. Not a very useful bot, but this is useful to test to see if everything is running smoothly. In order to test it, you can enter node . in your terminal and your bot should come online in Discord. Send a message to make sure your bot replies to you and that everything is flowing smoothly.

If it works, then great you are all done! Of course, there are way more things that can be done with the bot and I will show you some of the common ones. If you would like to skip ahead to the hosting, click here!

We went through replying to a message, but you can also have your bot send a message without replying using message.channel.send. You can even send a direct message to a user using message.author.send. If you want to check the contents of the message before doing anything, you can access the contents of the message using message.content. 


if (message.content == 'hello') {
    message.reply('Hello, how are you?');
}


It is important to keep your code neat so using a command handler can come in handy if you have multiple commands. Building on our previous index.js, we can change up our the client reacts to message creations. Create a new file called commands.js and a new folder called commands. In commands.js, place this. 


const burrito = require('./commands/burrito.js');
const help = require('./commands/help.js');

const commands = {
    burrito,
    help
};

const commandCharacter = "!";

module.exports = async function (message){
    
    try {
        if (message.content.charAt(1) ===" ") {
            return;
        }

        let tokens = message.content.split(" ");
        let command = tokens.shift();

        if (message.content.toLowerCase().includes("burrito")) {
            commands["burrito"](message, tokens);
        }
        
        if (command.charAt(0) === commandCharacter){
            //valid command
            command = command.toLowerCase().substring(1);
            commands[command](message, tokens);
        }
    }
    catch {
        return;
    }
}


Now, anytime a message is sent, the bot will first check if it is a valid message by scanning the first character that you can set to anything. A lot of bots use symbols such as ! or ? but you can use anything. You can also have specific cases to check in this section if you would like.

Anytime you create a new command, you must create a .js file for it in the commands folder and add it to the top with a require statement and place it in the commands array. An example of a simple reply command looks like this: 


module.exports = function (message, args) {
    if (!message?.author.bot) {
        message.reply('Yum, I love burritos!');
    }
}

If you want a more complex reaction system tied to roles within your server, you can use the following: 


const { MessageActionRow, MessageButton, MessageEmbed } = require('discord.js');

module.exports = async function (message) {
    try {
        const roleMapping = {
            '‚ù§Ô∏è': 'VIP',
            'üëç': 'Member',
            'üòé': 'Guest',
        };

        const sentMessage = await message.reply({
            content: 'React with an emoji to get a role:\n‚ù§Ô∏è - VIP\nüëç - Member\nüòé - Guest',
        });

        await sentMessage.react('‚ù§Ô∏è');
        await sentMessage.react('üëç');
        await sentMessage.react('üòé');

        const filter = (reaction, user) => !user.bot && Object.keys(roleMapping).includes(reaction.emoji.name);
        const collector = sentMessage.createReactionCollector({ filter, time: 60000, dispose: true });

        collector.on('collect', async (reaction, user) => {
            console.log(`Collected reaction: ${reaction.emoji.name} from ${user.tag}`);
            const roleName = roleMapping[reaction.emoji.name];
            console.log(`Role to be assigned: ${roleName}`);

            try {
                const role = message.guild.roles.cache.find(r => r.name === roleName);
                if (!role) {
                    console.error(`Role "${roleName}" not found.`);
                    return;
                }

                const member = await message.guild.members.fetch(user.id);

                if (member.roles.cache.has(role.id)) {
                    // await user.send(`You already have the "${roleName}" role.`);
                    return;
                }

                await member.roles.add(role);

                // await user.send(`You have been assigned the "${roleName}" role!`);
            } catch (error) {
                console.error('Error assigning role:', error);
            }
        });

        collector.on('remove', async (reaction, user) => {
            console.log(`Removed reaction: ${reaction.emoji.name} from ${user.tag}`);
            const roleName = roleMapping[reaction.emoji.name];
            console.log(`Role to be removed: ${roleName}`);

            try {
                const role = message.guild.roles.cache.find(r => r.name === roleName);
                if (!role) {
                    console.error(`Role "${roleName}" not found.`);
                    return;
                }

                const member = await message.guild.members.fetch(user.id);

                if (!member.roles.cache.has(role.id)) {
                    return;
                }

                await member.roles.remove(role);
                console.log(`Removed "${roleName}" role from ${user.tag}`);
            } catch (error) {
                console.error('Error removing role:', error);
            }
        });

        collector.on('end', (collected) => {
            if (collected.size === 0) {
                sentMessage.edit({ content: 'No reactions received, so no roles were assigned.' });
            } else {
                sentMessage.edit({ content: 'Roles have been assigned based on reactions!' });
            }
        });

    } catch (error) {
        console.error('Error sending message with role assignment:', error);
    }
};


Make sure the name of the roles is the same as written in roleMapping. And finally, you can create a reply that includes buttons that have their own features: 


const { ActionRowBuilder, ButtonBuilder, ButtonStyle } = require('discord.js');

module.exports = async function (message) {
    try {
        // Create the buttons using the new syntax
        const row = new ActionRowBuilder()
            .addComponents(
                new ButtonBuilder()
                    .setCustomId('yes')  // Custom ID for the 'Yes' button
                    .setLabel('Yes')     // Label to display
                    .setStyle(ButtonStyle.Success),  // Style for the button (green color)
                new ButtonBuilder()
                    .setCustomId('no')  // Custom ID for the 'No' button
                    .setLabel('No')     // Label to display
                    .setStyle(ButtonStyle.Danger), // Style for the button (red color)
                new ButtonBuilder()
                    .setCustomId('maybe')  // Custom ID for the 'Maybe' button
                    .setLabel('Maybe')     // Label to display
                    .setStyle(ButtonStyle.Secondary) // Style for the button (gray color)
            );

        // Send the message with buttons
        const sentMessage = await message.reply({
            content: 'Please choose an option:',
            components: [row],  // Attach the row with buttons to the message
        });

        // Create a message collector to listen for button interactions
        const filter = interaction => interaction.isButton() && interaction.message.id === sentMessage.id;
        const collector = sentMessage.createMessageComponentCollector({ filter, time: 15000 });  // Collect interactions for 15 seconds

        // Handle button interactions
        collector.on('collect', async (interaction) => {
            if (interaction.customId === 'yes') {
                await interaction.reply({ content: 'You selected Yes!', ephemeral: true });
            } else if (interaction.customId === 'no') {
                await interaction.reply({ content: 'You selected No!', ephemeral: true });
            } else if (interaction.customId === 'maybe') {
                await interaction.reply({ content: 'You selected Maybe!', ephemeral: true });
            }
        });

        // Handle time expiration
        collector.on('end', collected => {
            if (collected.size === 0) {
                sentMessage.edit({ content: 'You did not select any option.', components: [] });
            } else {
                sentMessage.edit({ content: 'Thanks for your response!', components: [] });
            }
        });

    } catch (error) {
        console.error('Error sending buttons:', error);
    }
};


A note for this code, if ephemeral is set to true, that means that only the user who entered the command will see the message. Currently, this code only replies to the user with their response but you can go as deep as you want and implement your own functions for each option. You can even edit how long the messages stay active. You are free to do as you wish.

Now, for hosting there are two options: Locally, and online. For local it is simple, you just run the node . command and it is done. You do have to keep vscode open for this to work but if you have an extra machine that stays on or you do not mind keeping it open, this is the easiest and simplest way.

If you prefer online hosting, there are many options but the free options are limited. Most online hosting sites will have free tiers that you can use but will charge you if you pass them. Google Cloud Console/App Engine is an option. There are many online tutorials and videos to help set that up. Another option is Cloudflare which Discord has an official tutorial for setting it up.

In the end, it is almost impossible to find a perfectly 100% free solution online but if your bot is on the lower end and will only be in a few servers, the online free options should be enough. Just make sure to keep an eye on them and that you do not get charged!

And that's it! If you made it this far, I hope you were able to successfully create your bot and found a reasonable way for you to host it. If you need any more help, there are many online videos and documentation that can help, or you can leave a comment down below!
